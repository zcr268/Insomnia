Keep your PC awake
when it matters

Insomnia prevents Windows from sleeping — automatically when AI coding tools are working, when specific apps are running, or manually on demand. No more broken sessions.

↓ Download for Windows View on GitHub

You're running Claude Code or Aider on a long task. You step away. Your PC falls asleep, kills the process, and you come back to a broken session and lost work. Setting your sleep timeout to "Never" works — but then your PC never sleeps when you actually want it to.

Insomnia fixes this. It stays awake only while something needs it, then gets out of the way.

Features

Smart, not blunt

✨

Hook-based AI integration

For Claude Code, Insomnia hooks directly into the event system — awake while Claude is working, asleep the moment it stops. Precise to the second.

📋

App watching

Add any app as a trigger. Insomnia auto-discovers every installed app on your system. When it's running, you stay awake. When it closes, you don't.

🔘

Manual toggle

Need to stay awake right now? One switch. No timers, no fuss. Toggle it off when you're done.

🦉

Lives in the tray

A purple owl means awake. A grey owl means inactive. The hover tooltip tells you exactly why. Close the window — it keeps running.

⚡

Zero bloat

Plain HTML, CSS, and JS. No React. No bundler. Zero external runtime dependencies. Installs in seconds and uses almost no resources.

🔒

Local and private

Nothing leaves your machine. No telemetry, no accounts, no cloud. Settings are stored locally and that's it.

Integrations

Built for AI-assisted coding

Tool Type How it works
✨ Claude Code Smart Hook-based — awake only while Claude is actively running tools. Idle = asleep.
✏️ Cursor Process Awake while Cursor is open.
🤖 Aider Process Awake while Aider is running.
💻 OpenAI Codex Smart Hook and session-activity based — awake while Codex is actively working in CLI, VS Code, or the standalone app.
🧠 Ollama Process Awake during local model inference.
